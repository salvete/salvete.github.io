预览

TCP协议保证可靠传输，但是网络只能尽力传送，为了解决二者之间的矛盾，我们需要做一些工作。这些工作可以使我们达到我们的目标。

接下来的几个实验的大致功能：

  • lab1:完成数据的组装，使无序的字节流变为有序
  • lab2:实现TCP的接收端
  • lab3:实现TCP发送端
  • lab4:实现TCP的连接

  • git merge origin/lab1-startercode

需要实现的接口如下：

翻译一下就是：

  • StreamReassembler(const size_t capacity)：构造函数，capacity为容量。注意：这里的容量是指数据流缓冲区的大小，而不是该StreamReassembler所能够使用的内存大小（我因为理解题意出错而考虑得特别复杂，导致始终有三个点无法通过）。实际上，StreamReassembler内部自定义的数据结构可以使用的空间没有限制。
  • void push_substring(const string &data, const uint64_t index, const bool eof)：这是最关键的函数，完成写入操作。需要考虑三种情况：不相交，相交，覆盖。
  • size_t unassembled_bytes() const：还没有被组装的字节个数，如果同一位置的字节发送了多次，那么算作一次。
  • bool empty() const：是否完成。

遇到的问题

  • 未充分理解题意，导致问题复杂化。
  • 判断完成的条件是eof片段出现并且组装数等于总数，如果不判断eof片段出现，那么遇到这种情况会出错：push_substring(“”,0,false)。此时assembled_num == total_num,但是并没有结束。
  • 判断覆盖的时候，应该使用>而不是>=。当然可以通过添加其他判断条件解决，这只是一种解决方式。
