静态域与静态方法

静态域

静态域属于整个类，但是对象可以对其进行操作。

静态方法

静态方法不能访问实例域，但可以访问静态域。实例方法既可以访问实例域，也可以访问静态域。静态可以理解为：属于类且不属于类对象的函数和变量。

工厂方法

可以通过使用工厂方法来完成构造器的工作。这样做有两个原因：

  • 无法命名构造器。构造器必须与类名相同。
  • 当使用构造器的时候，无法改变所构造的对象类型。

例子：

  • LocalDate.now(),其中LocalDate为类名。
  • LocalDate.of(),同上。
main方法

每一个类可以有一个main方法，可以使用这个方法对该类进行测试。

对象构造

对象的构造有三种方法。

  • 构造器
  • 声明时赋值
  • 初始化块

调用构造器的处理步骤：

  1. 所有数据域被初始化为默认值（0,false或null）。
  2. 按照在类中声明的顺序，依次执行所有域初始化语句和初始化块。
  3. 如果构造器第一行调用了第二个构造器，则执行第二个构造器。
  4. 执行这个构造器的主体。

类、超类和子类

继承

在Java中，所有的继承都是公有继承，而没有C++里面的私有继承和保护继承。

子类构造器

子类构造器中使用super调用构造器的语句必须是子类构造器的第一条语句。

多态

置换法则：程序中出现超类对象的任何地方都可以使用子类对象置换。

理解方法调用

假若调用x.f(args)，其中x是一个类C的对象。下面为过程调用的描述：

  • 编译器查看对象的声明类型和函数名。编译器会一一列举所有C类中名为f的方法和其他超类中访问属性为public且名为f的方法(超类的私有方法不可访问)。
  • 接下来，编译器会查看调用方法时提供的参数类型。这个过程称为重载解析：如果存在名为f且参数与提供的参数完全匹配的方法，就选择这个方法。否则，报告一个错误。
  • 如果是private、static、final方法或者是构造器，那么很自然地可以找到相应的调用，这种称为静态绑定。与此对应的是动态绑定。
  • 如果是动态绑定，虚拟机一定调用与x所引用对象的实际类型最合适的那个类的方法。假设x的实际类型是C，它是B的子类。如果C中定义了f(args)，则直接调用；否则在C的超类B中寻找f(args)，以此类推。

每次调用都要搜索，时间开销颇大，因此虚拟机为每个类都创建了一个方法表。每次调用，仅需要查找这个表就行了。另外，覆盖一个方法的时候，子类方法不能低于超类方法的可见性。将一个类声明为final，只有其中的方法自动地成为final，而不包括域。

强制类型转换
  • 只能在继承层次内进行类型转化
  • 在将超类转换为子类之前，应该使用instanceof进行检查
抽象类

为了提高程序的清晰度，包含一个或多个抽象方法的类本身必须被声明为抽象的。许多程序员认为，在抽象类中不能包含具体的方法。建议将通用的域和方法（不管是否是抽象的）放在超类中（不管超类是否是抽象的）。抽象方法充当占位的角色，它的具体实现在子类中。抽象类不能被示例化。

Java中的访问修饰符
  • 仅对本类可见–private
  • 对所有类可见–public
  • 对本包和子类可见–protected
  • 对本包可见–默认，不需要修饰符

Object类

equals方法

编写完美equals方法的建议：

  1. 显式参数命名为otherObject,稍后需要将其转换为另一个叫做other的变量

  2. 检测this和otherObject是否指向同一个对象：if (this == otherObject) return true;

  3. 检测otherObject是否为null: if (otherObject == null) return false;

  4. 比较this和otherObject是否属于同一类。若equals的语义在每个子类中有所改变，就使用getClass()检测：

    if (getClass() != otherObject.getClass()) return false; ;若所有子类都有统一的语义，那么就使用instanceof检测：if (!(otherObject instanceof ClassName)) return false;

  5. 将otherObject转换为相应的类类型的变量：ClassName other = (ClassName)otherObject;

  6. 现在开始对所有需要比较的域进行比较。使用 ==对基本类型进行比较；使用 equals比较对象域。如果所有域都匹配，就返回true;否则返回false

如果子类中重新定义equals，就要在其中包含super.equals(other)或者super.equals( (SuperClassName) other),在上层确定二者相同，而后在此层比较。需要注意的是步骤4中所选用的检测方法：

  • 如果子类能够拥有自己的相等概念，则对称性需求将强制采用getClass进行检测。
  • 如果超类决定相等的概念，那么就可以使用instanceof进行检测，这样可以在不同子类的对象之间进行相等了的比较。
hashCode方法

equals与hashCode方法的定义必须一致：若equals返回true，那么两对象的hashCode应该是相同的。
